# Simple RAG (Retrieval-Augmented Generation) Project

## Обзор

Этот проект представляет собой обучающий пример реализации системы Retrieval-Augmented Generation (RAG) на Python. Проект создан с целью помочь новичкам в области искусственного интеллекта и обработки естественного языка понять основы работы RAG-систем, которые объединяют возможности языковых моделей с извлечением релевантной информации из базы знаний.

## Что такое RAG?

RAG (Retrieval-Augmented Generation) — это подход, при котором генерация текста языковой моделью улучшается за счёт предварительного поиска релевантной информации во внешней базе знаний. Это позволяет:

1. Улучшить точность ответов модели
2. Предоставлять актуальную информацию без необходимости переобучения модели
3. Снизить проблему галлюцинаций (выдумывания фактов) у языковых моделей
4. Работать с данными, которые отсутствовали в обучающем наборе модели

## Используемые технологии

- **Python** — основной язык программирования
- **FAISS** — библиотека для эффективного векторного поиска
- **Sentence Transformers** — создание эмбеддингов для текстов
- **BM25** — алгоритм ранжирования текстовых документов
- **Pydantic** — валидация данных и создание конфигураций
- **Python-dotenv** — работа с переменными окружения

## Начало работы

### Предварительные требования

- Python 3.8+ 
- Доступ к интернету для загрузки моделей (при первом запуске)

### Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/simple_rag.git
   cd simple_rag
   ```

2. Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # для Linux/Mac
   # или
   venv\Scripts\activate  # для Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` в корне проекта и укажите необходимые переменные окружения (если требуется):
   ```
   MODEL_PATH=model/your-model-name
   # Другие настройки
   ```

### Запуск

Запустите основной файл:
```bash
python main.py
```

## Как это работает

1. **Предварительная обработка данных**:
   - Загрузка текстовых файлов из директории `data/`
   - Разбиение текста на фрагменты (chunks) с помощью модуля `splitters.py`

2. **Индексация и векторизация**:
   - Преобразование текстовых фрагментов в векторы с помощью Sentence Transformers
   - Создание эффективного FAISS-индекса для быстрого поиска

3. **Поиск и ранжирование**:
   - Преобразование запроса пользователя в вектор
   - Поиск ближайших векторов с помощью FAISS
   - Переранжирование результатов с помощью алгоритма BM25

4. **Генерация ответа**:
   - Формирование контекста из найденных фрагментов
   - Генерация ответа с помощью языковой модели и шаблона промпта


## Расширение функциональности

Проект специально создан модульным, чтобы вы могли:
- Добавлять свои источники данных в директорию `data/`
- Экспериментировать с разными моделями для создания эмбеддингов
- Настраивать и оптимизировать параметры поиска
- Подключать различные языковые модели для генерации ответов

## Часто задаваемые вопросы

### Какие требования к оборудованию?
Для базовой работы достаточно компьютера с 8+ ГБ RAM, но для работы с более крупными языковыми моделями может потребоваться мощный GPU.

### Можно ли использовать локальные LLM?
Да, проект поддерживает работу с локальными языковыми моделями через модуль `utils/llm_client.py`.

### Как добавить свои данные?
Просто добавьте текстовые файлы в директорию `data/` и перезапустите индексацию.
